# Circuit-Breaker Inconsistency — 2026-05-14

## Summary
During the market_open routine on 2026-05-14, the circuit-breaker advanced from **OUT → FULL** (via HALF) based on Alpaca account equity, then was written back to **OUT** by a second pass using paper_sim equity. Final state is **OUT**.

## Root cause
Two separate equity sources were used in the same routine run:
1. **Alpaca account equity** ($135,029.81) — used in the first advance() call, which reset the inflated peak and advanced state to FULL.
2. **paper_sim equity** ($100,605.39) — used in a second write, which reverted to the original peak ($119,140.25) and recomputed state as OUT (15.58% DD).

The history file records only the first two transitions; the OUT reversion at 13:41Z was a **direct write** that bypassed advance() and did not append to history.

## Inconsistency
`circuit_breaker_history.jsonl` shows FULL as last advance() output, but `circuit_breaker.json` shows OUT. These are now out of sync.

## Recommended follow-up
- Standardise the CB equity source: always use `paper_sim.portfolio_equity()` (cash + MTM of positions.json), not the Alpaca account balance (which includes cash movements unrelated to our paper positions).
- Add a reconcile check between history and JSON at the start of each routine.
- Track in `prompts/proposed_updates/` as lesson-pending item.

## Severity: MEDIUM (data integrity)
Trading decisions are not impacted (state=OUT is the conservative choice), but the inflated-peak artifact remains unresolved and the dual-source pattern is a recurring failure mode.
